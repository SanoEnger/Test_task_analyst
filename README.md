# Тестовое задание на позицию Fullstack-аналитика

Решение комплексного тестового задания, включающее в себя моделирование бизнес-процессов (BPMN), системный анализ требований (User Story, Use Case) и проектирование технической архитектуры (REST API OpenAPI, алгоритмы бэкенда).

## 📂 Структура репозитория

* **`[task_1/](https://github.com/)`** — Модель бизнес-процесса выдачи ИТ-оборудования в нотации BPMN 2.0. Содержит исходный файл `.bpmn` и экспортированное изображение `.png`. Также внутри описаны ключевые системные противоречия процесса "As-Is".
* **`[task_2/](https://github.com/)`** — Описание функциональности Личного кабинета продавца (Публикация товара) в формате User Story и подробного Use Case со всеми альтернативными сценариями.
* **`[task_3/](https://github.com/)`** — Проектирование REST API для регистрации пользователя. Содержит спецификацию в формате OpenAPI 3.0 (`openapi.yaml`), примеры JSON-запросов/ответов, а также пошаговый алгоритм работы бэкенд-сервиса с учетом валидации и безопасности.

## 🛠️ Используемые инструменты и технологии
* **Моделирование процессов:** BPMN 2.0 (bpmn.io / Camunda Modeler)
* **Проектирование API:** OpenAPI 3.0 / Swagger
* **Документация:** Markdown
